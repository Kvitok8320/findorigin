# Настройка API Яндекс.Поиска для сайта (правильный способ)

## Важно!

**Yandex Cloud Search API** — это другой сервис, не то, что нам нужно.

Нам нужен **"API Яндекс.Поиска для сайта"** из Кабинета разработчика Яндекса.

## Пошаговая инструкция:

### Шаг 1: Получение ключа авторизации

1. Перейдите на **https://developer.tech.yandex.ru**
2. Войдите в свой Яндекс аккаунт
3. Нажмите кнопку **"Получить ключ"** или **"Создать ключ"**
4. Заполните форму:
   - **Имя ключа**: например, "FindOrigin Bot"
   - **Сервис для подключения**: выберите **"API Яндекс.Поиска для сайта"**
   - Согласитесь с условиями использования
5. Нажмите **"Отправить"** или **"Создать"**
6. **Скопируйте полученный ключ** (показывается только один раз!)

### Шаг 2: Создание поиска для сайта (если нужно)

1. Перейдите на **https://site.yandex.ru/searches/**
2. Если у вас еще нет поиска, создайте новый
3. На странице поиска найдите раздел **"Выдача в JSON"**
4. В поле **"API-ключ"** введите полученный ключ из шага 1
5. Сохраните изменения

**Примечание:** Изменения вступят в силу в течение часа.

### Шаг 3: Настройка в Vercel

1. Зайдите на https://vercel.com
2. Откройте ваш проект
3. Перейдите в **Settings** → **Environment Variables**
4. Добавьте переменную:
   - **Key**: `YANDEX_API_KEY`
   - **Value**: ваш ключ из шага 1
5. Нажмите **"Save"**
6. **Передеплойте проект** (Redeploy)

### Шаг 4: Тестирование

1. Отправьте сообщение боту в Telegram
2. Бот должен найти источники через Яндекс.Поиск API

## Важные ограничения:

- **Ключ можно использовать только для одного поиска**
- Если нужны разные поиски, создайте отдельные ключи для каждого
- Попытка подключить один ключ к двум и более поискам вызовет ошибку

## Документация:

- Кабинет разработчика: https://developer.tech.yandex.ru
- API Яндекс.Поиска для сайта: https://yandex.ru/dev/site/api/
- Мои поиски: https://site.yandex.ru/searches/

## Отличие от Yandex Cloud Search API:

- **API Яндекс.Поиска для сайта** — простой API для поиска на сайте, не требует Yandex Cloud
- **Yandex Cloud Search API** — сложный сервис для корпоративных решений, требует настройки в Yandex Cloud

Для нашего бота нужен первый вариант — **API Яндекс.Поиска для сайта**.

# Как проверить логи на Vercel

## Пошаговая инструкция:

### Шаг 1: Откройте проект на Vercel

1. Зайдите на https://vercel.com
2. Войдите в свой аккаунт
3. Найдите и откройте проект **FindOrigin**

### Шаг 2: Откройте раздел Deployments

1. В меню слева или вверху найдите **"Deployments"** (Деплои)
2. Нажмите на него
3. Вы увидите список всех деплоев

### Шаг 3: Выберите последний деплой

1. Найдите самый последний (верхний) деплой
2. Нажмите на него (или на три точки рядом с ним)

### Шаг 4: Откройте Functions

1. В открывшемся окне деплоя найдите раздел **"Functions"** (Функции)
2. Или вкладку **"Functions"** вверху
3. Вы увидите список функций:
   - `/api/telegram`
   - `/api/test-analyze`
   - `/api/webhook/set`

### Шаг 5: Откройте логи функции `/api/telegram`

1. Найдите в списке `/api/telegram`
2. Нажмите на неё
3. Откроется страница с логами выполнения этой функции

### Шаг 6: Проверьте логи

Вы увидите:
- **Invocations** (Вызовы) - когда функция вызывалась
- **Logs** (Логи) - что происходило при выполнении
- **Errors** (Ошибки) - если были ошибки

## Что искать в логах:

### ✅ Хорошие признаки:
- Видны вызовы функции (Invocations)
- Нет ошибок в разделе Errors
- В логах есть записи о получении update

### ❌ Проблемы:
- **"TELEGRAM_BOT_TOKEN is not set"** - не настроена переменная окружения
- **"Error in async update processing"** - ошибка при обработке сообщения
- **"Telegram API error"** - проблема с отправкой сообщения
- Нет вызовов функции - webhook не работает или не получает обновления

## Альтернативный способ через браузер:

1. Зайдите на https://vercel.com/dashboard
2. Выберите проект
3. В меню слева: **Deployments**
4. Нажмите на последний деплой
5. Вверху вкладки: **Functions**
6. Нажмите на `/api/telegram`
7. Смотрите логи

## Скриншот того, что должно быть:

```
Deployments → [Ваш деплой] → Functions → /api/telegram
```

Там вы увидите:
- Список вызовов (Invocations)
- Логи каждого вызова
- Ошибки, если они есть
